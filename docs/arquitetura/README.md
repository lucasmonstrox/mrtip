# docs/arquitetura

Decisões técnicas cross-feature que valem além de um plano único (`<slug>.md`) — o que `/rs` e `/pl` **não re-decidem**, só detalham. Ex.: separação quant/LLM da camada de IA, onde rodam os coletores de ingestão, escolha de banco/persistência, contrato do "dossiê do jogo".

Distinto de `docs/visao-geral.md` (produto/por quê) e de `docs/planos/` (como fazer uma feature). Aqui mora o **como o sistema é**, transversal. Decisão cravada aqui manda sobre investigação/plano posterior.
